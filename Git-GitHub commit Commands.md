## Summary →

feat: add new feature

fix!: breaking change bug fix

fix: bug fix

refactor: improve code structure

perf: improve performance

style: UI changes

test: add/update tests

chore: maintenance tasks

feat!: breaking change

docs: documentation update

# ✅ 1. New Feature (Feature Addition)

When you add something new.

### Format:

```
feat: short description of feature
```

### Examples:

- `feat: add user login API`
- `feat: implement payment gateway integration`
- `feat: add dark mode toggle`

---

# ✅ 2. Bug Fix

When you fix a problem.

### Format:

```
fix: short description of bug fix
```

### Examples:

- `fix: resolve login crash on invalid password`
- `fix: correct navbar alignment issue`
- `fix: handle null response in API`

---

# ✅ 3. Code Refactor (No feature change)

Improving code without changing behavior.

### Format:

```
refactor: what you improved
```

### Examples:

- `refactor: simplify authentication logic`
- `refactor: optimize database queries`
- `refactor: restructure project folders`

---

# ✅ 4. Performance Improvement

When you make things faster or lighter.

### Format:

```
perf: description
```

### Examples:

- `perf: reduce API response time`
- `perf: optimize image loading`

---

# ✅ 5. UI/Design Changes

Frontend visual updates.

### Format:

```
style: description
```

### Examples:

- `style: update login page UI`
- `style: improve button spacing`

---

# ✅ 6. Documentation Update

Only docs changed.

### Format:

```
docs: description
```

### Examples:

- `docs: update README with setup steps`
- `docs: add API usage guide`

---

# ✅ 7. Test Changes

Adding or updating tests.

### Format:

```
test: description
```

### Examples:

- `test: add unit tests for auth service`
- `test: improve login API test coverage`

---

# ✅ 8. Chore (maintenance tasks)

Build tools, dependencies, config.

### Format:

```
chore: description
```

### Examples:

- `chore: update dependencies`
- `chore: configure ESLint`

---

# 🚀 9. Breaking Change (Important!)

When old code will break.

### Format:

```
feat!: description
```

or

```
fix!: description
```

### Example:

- `feat!: change authentication flow (BREAKING CHANGE)`
